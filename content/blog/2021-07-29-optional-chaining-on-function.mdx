Just notice the optional chaining `?.` can also apply to `Function`.

For example,

```ts
const echo = (name: string): string => name;

echo?.('hi'); // 'hi'
```

which means that `echo?.('hi')` is equal to `echo ? echo('hi') : void`

The use case I saw is from the npm package [react-error-boundar](https://github.com/bvaughn/react-error-boundary/blob/master/src/index.tsx)
